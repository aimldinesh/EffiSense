## EffiSense: End-to-End MLOps Pipeline for Machine Efficiency Prediction

EffiSense is a complete MLOps pipeline that predicts the **efficiency status (High | Medium | Low)** of industrial machines using intelligent manufacturing data.

It features a fully automated CI/CD setup using **Jenkins**, **ArgoCD**, **Docker**, **Kubernetes**, and **GitHub Webhooks** for seamless deployment.

## üìä Dataset Information

It is designed to support research in areas such as **predictive maintenance**, **quality control**, **resource optimization**, and **real-time anomaly detection** within Industry 4.0 and 6G-enabled smart factories.


### üìÅ Data Source

This is a simulated dataset representing intelligent manufacturing systems, originally hosted on **Kaggle** (by Ziya). It mimics real-world industrial sensor streams and network metrics for research and development use.

## ü§ñ Model Details

EffiSense applies a supervised ML model to classify machine efficiency as **High**, **Medium**, or **Low** using preprocessed industrial data.

### üß† Problem Type
- Multiclass Classification

### üßÆ ML Pipeline Overview

1. **Data Preprocessing**
   - Encode categorical features
   - Scale numeric features using `StandardScaler`
   - Split dataset into training and test sets (80/20)

2. **Model Training**
   - Algorithm: **Logistic Regression** (scikit-learn)
   - Input: Preprocessed `X_train` and `y_train`
   - Output: `model.pkl` (final trained model)

3. **Model Evaluation**
   - Evaluated on `X_test`, `y_test`
   - Metrics: Accuracy, Precision, Recall, F1-score
   - Results saved to `evaluation_metrics.csv`

## üöÄ CI/CD Pipeline

EffiSense features a fully automated **CI/CD pipeline** that ensures fast, reliable, and reproducible ML model deployment. It leverages:

- üß™ **Jenkins** for Continuous Integration (CI)  
- üîÅ **ArgoCD** for GitOps-based Continuous Deployment (CD)  
- üåê **GitHub Webhooks** for automatic trigger on every push  

Together, these tools enable true MLOps automation ‚Äî from commit to deployment ‚Äî using Docker and Kubernetes.


### ‚öôÔ∏è 1. Development & Experimentation

> The ML workflow starts with experimentation, model building, and evaluation ‚Äî all locally.

- üß± **Project Setup**: Organize modular folder and script structure  
- üß™ **Notebook Testing**: Validate core logic in Jupyter  
- üßπ **Data Processing**: Preprocess raw data (encoding, scaling, splitting)  
- üß† **Model Training**: Train Logistic Regression and evaluate metrics  


### üì¶ 2. Packaging & Infrastructure

> The system is containerized and made deployment-ready.

- üíª **User App Building**: Develop Flask web UI for predictions  
- üê≥ **Dockerization**: Define `Dockerfile` to containerize the app  
- üìÑ **Kubernetes Manifests**: Write `deployment.yml` and `service.yml`  
- üìÅ **Versioning**: Store models and scaler as `.pkl` inside `artifacts/`  
- ‚òÅÔ∏è **VM Setup**: Use **GCP VM** (Ubuntu 20.04) with Minikube + kubectl  


### üîÑ 3. CI/CD Pipeline Automation

> The complete automation is triggered via GitHub push and executed in Jenkins and ArgoCD.

- üîß **Jenkins Setup**: Installed and configured on GCP VM  
- üîó **GitHub ‚Üî Jenkins Integration**: GitHub Webhooks trigger CI on push  
- üõ†Ô∏è **CI Pipeline** (`Jenkinsfile`):  
  - Pull latest code  
  - Build Docker image  
  - Run lint/test scripts  
  - Push image to DockerHub/GCR  
- üöÄ **ArgoCD Setup**: Installed in Kubernetes cluster via Minikube  
- üîÅ **CD Process**:  
  - ArgoCD monitors Git for manifest changes  
  - On detecting a new image or config update, it syncs automatically  
  - App is deployed to Kubernetes within seconds  


### üîî CI/CD Trigger Summary

> ‚úÖ **Trigger:** A `git push` to the `main` branch activates the Jenkins CI pipeline via webhook.  
> üîÅ **Deployment:** Updated Kubernetes manifests are pulled and deployed by ArgoCD automatically.
